✅ Request Community Feature - Complete Setup

What Was Fixed

Problem: The "Request Community" button was showing a page that said "You have no pending community join requests" instead of allowing users to request to create a new community.

Solution: Created a complete community request system that allows regular users to submit requests to create communities, which admins can later review and approve.

How It Works Now

For Regular Users (Non-Admin):

  1. Click the "Request Community" button
  2. Fill out the form with:
    • Community Name (required)
    • Community URL (required)
    • Description (required)
    • Community Type (Public/Private)
    • Category
    • Sub-category (if applicable)
    • Reason for Request (required) - explain why you want to create this community
  3. Submit the form
  4. Request is saved with status "pending"
  5. Admin will review and approve/reject the request

For Admins/Moderators:

  1. Admins see the "Create Community" button instead (can create directly)
  2. View pending requests in database: Wo_Community_Requests table
  3. Approve requests manually (admin panel integration coming soon)

Setup Instructions

⚠️ IMPORTANT: You MUST run this setup before testing!

Step 1: Create Database Table

Visit this URL in your browser:

http://localhost/facesofnaija/setup_community_requests.php

This will automatically create the Wo_Community_Requests table.

Step 2: Clear Browser Cache

Press Ctrl+Shift+Delete or Ctrl+F5 to load the new templates and scripts.

Step 3: Test the Feature

  1. Log in as a regular user (not admin)
  2. Go to Communities section
  3. Click "Request Community"
  4. Fill out and submit the form
  5. You should see a success message

Files Modified

1. Template File

themes/facesofnaija/layout/community/request-community.phtml

Changed from: Viewing pending join requests
Changed to: Form to request community creation

2. XHR Handler

xhr/communities.php

Added: request_community handler that:

  • Validates form data (same validation as create community)
  • Checks if user is admin (admins should use Create button)
  • Saves request to Wo_Community_Requests table
  • Returns success/error messages

3. Database Table

Wo_Community_Requests - Stores all community creation requests

Columns:

  • id - Auto-increment primary key
  • user_id - Who submitted the request
  • community_name - Requested community URL name
  • community_title - Requested community title
  • about - Community description
  • category - Community category ID
  • sub_category - Sub-category (optional)
  • privacy - 1=Public, 2=Private
  • reason - User's reason for requesting
  • status - 'pending', 'approved', or 'rejected'
  • time - When request was submitted
  • reviewed_by - Admin ID who reviewed (null if pending)
  • reviewed_at - When it was reviewed (null if pending)
  • admin_notes - Admin's notes about the request

Database Queries for Admins

View All Pending Requests:

SELECT r.*, u.username, u.email 
FROM Wo_Community_Requests r
JOIN Wo_Users u ON r.user_id = u.user_id
WHERE r.status = 'pending'
ORDER BY r.time DESC;

Approve a Request (Example - ID 1):

-- First, update the request status
UPDATE Wo_Community_Requests 
SET status = 'approved', 
    reviewed_by = YOUR_ADMIN_USER_ID, 
    reviewed_at = UNIX_TIMESTAMP()
WHERE id = 1;

-- Then create the community using data from the request
-- (Copy the data and use the Create Community admin panel)

Reject a Request:

UPDATE Wo_Community_Requests 
SET status = 'rejected', 
    reviewed_by = YOUR_ADMIN_USER_ID, 
    reviewed_at = UNIX_TIMESTAMP(),
    admin_notes = 'Reason for rejection'
WHERE id = 1;
